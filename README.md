# Paksh Khattar GitHub Profile README Design

## Objective

Replace the existing README in `paksh1/Paksh1` with a concise, animated, playful, and recruiter-friendly GitHub profile. The profile will position Paksh primarily as a Penetration Tester and secondarily as a Security Engineer.

## Audience

- UK recruiters hiring for junior penetration-testing and security-engineering roles
- Cybersecurity practitioners and open-source collaborators
- Developers interested in security automation and secure software

## Visual Direction

- Dark graphite and near-black foundation
- Neon-green accents only; no pink, purple, blue, or multicolour decoration
- `PAKSH // ROOT LAB` hero inspired by a terminal or hacker lab
- Professional body layout with generous spacing and limited badges
- Animation delivered through GitHub-compatible SVG services, with readable fallback text

## Content Structure

1. Animated hero with Paksh's name, primary roles, London location, CEH v13, and open-to-work status
2. Short positioning statement combining offensive security and engineering discipline
3. Current-focus terminal block
4. Mission highlights:
   - AI Bug Hunter
   - Pathfinder OSINT
   - CareerCanvas
5. Grouped operator toolkit:
   - Offensive Security
   - Security Operations
   - Engineering
   - Cloud and Data
6. Collapsible `Classified Lab Notes` area for current learning, practice platforms, and collaboration interests
7. GitHub statistics and activity widgets styled to match the dark neon-green palette
8. Clickable GitHub, LinkedIn, and email calls to action

## Interaction and Animation

- Animated header and typing line
- Clickable contact badges and verified repository links
- Native HTML `<details>` elements for collapsible sections
- Dynamic GitHub statistics
- No custom JavaScript, because GitHub profile Markdown does not permit it

## Evidence and Accuracy

- Claims will be limited to information supported by the five supplied résumés
- Public repository links will be verified before inclusion
- Private or unavailable projects will be described without broken links
- The public phone number will not be included
- The existing duplicated statistics and oversized unverified skill wall will be removed

## Delivery and Validation

- Deliver one standalone `README.md`
- Validate Markdown structure, HTML tag balance, URLs, username parameters, and absence of placeholders
- Confirm the file contains no unsupported scripts, private data, or unverified claims
